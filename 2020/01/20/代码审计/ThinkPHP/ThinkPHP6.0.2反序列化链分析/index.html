ThinkPHP6.0 RCE反序列化链分析

环境搭建

  • PHP 7.2.14
  • ThinkPHP 6.0.2

开启ThinkPHP6调试：将根目录.example.env更改为.env，文件中添加：APP_DEBUG = true

漏洞挖掘

搬运一个利用链：

POP链类似于ThinkPHP5.1，首先全局搜索：__desctruct()

在vendor/topthink/think-orm/src/Model.php的Model中找到

$this->lazySave可控，赋值为True，往下查看save()

关键要执行$this->updateData()函数，要执行该函数的条件显然可控：

  1. $this->isEmpty() == false
  2. true === $this->trigger(‘BeforeWrite’)
  3. $this->exists == true

完成前置条件后继续往下看updateData()，这里关键要执行checkAllowFields()，

要执行checkAllowFields()必须满足前面条件：

  1. $this->trigger(‘BeforeUpdate’) != false

  2. empty($data) != false

  3. $data值来源于getChangedData()：应使$this->force为true，直接返回$data，否则会返回其他干扰值。

继续往下checkAllowFields()，轻易满足前面条件，最终执行$this->db()

查看$this->db()源码：

$this->connection可控，赋值为”mysql”；name()方法参数完全可控，可触发__toString()

后面POP链与ThinkPHP5.1相同，首先触发Conversion类中的__toString()方法

往下看toArray()中：这里需执行最后一个getAttr()方法，条件可控

getAttr()位于vendor/topthink/think-orm/src/model/concern/Attribute.php中

经过分析发现$value为执行命令，经过getData()处理后返回值，getData()也可控

命令执行点在getValue()，和ThinkPHP5.1.35类似，不再赘述。

最终执行的是system([命令],$this->data)，这样的格式是允许的，要求第二个参数是某变量的引用即可。
